# Hook 系统设计方案

日期：2026-06-21  
项目：`learn-agent`  
文档状态：待实现

## 1. 背景

当前项目中，agent loop 的所有行为逻辑（工具调用、结果处理、上下文压缩、循环退出）都硬编码在 `agent_loop.py` 中。随着功能增长（compaction、日志、监控、安全控制），这种耦合方式会越来越难维护。

Hook 系统提供一种**可扩展的拦截机制**，允许外部代码在 agent 生命周期的关键节点注入自定义逻辑，而无需修改核心 loop 代码。

## 2. 目标

1. 提供 6 个生命周期的 hook 阶段，覆盖 agent 从启动到退出的完整流程。
2. Hook 天然全局生效——父 agent 和子 agent 共享同一套 hook，通过 `config.role` 自行区分。
3. Hook 可以**修改上下文数据**（如 tool_result、tool_input）。
4. Hook 可以**拦截工具执行**（PreToolUse 返回 False = 阻止工具调用）。
5. 注册方式为代码注册（`hook_registry.register()`），参考 `register_tools` 的设计模式。
6. 失败隔离：单个 hook 异常不影响其他 hook 和主流程。
7. 支持 priority 排序，priority 越小越先执行。

## 3. 非目标

1. 不支持配置文件注册 hook（MVP 仅代码注册）。
2. 不重构现有 compaction 为 hook 实现（后续验证稳定后再迁移）。
3. 不支持 hook 的异步执行。
4. 不支持 hook 间的依赖关系。
5. 不支持热加载/卸载 hook。

## 4. 整体设计

### 4.1 六个 Hook 阶段

| 阶段 | 触发位置 | 触发时机 | data 字段 |
|------|----------|----------|-----------|
| `USER_PROMPT_SUBMIT` | `main.py`，`agent_loop()` 前 | 用户输入 prompt 后 | `prompt` |
| `PRE_TOOL_USE` | `agent_loop.py`，`execute_tool_use_blocks()` | 工具 handler 执行前 | `tool_name`, `tool_input` |
| `POST_TOOL_USE` | `agent_loop.py`，`execute_tool_use_blocks()` | 工具 handler 返回后 + L1 之后 | `tool_name`, `tool_input`, `result` |
| `SUBAGENT_START` | `run_delegate_task.py`，`agent_loop()` 前 | 子 agent 派发前 | `task`, `context`, `relevant_paths` |
| `SUBAGENT_STOP` | `run_delegate_task.py`，`agent_loop()` 后 | 子 agent 返回后 | `result`, `child_state` |
| `STOP` | `agent_loop.py`，统一退出点 | agent loop 退出前 | `reason` |

### 4.2 注入点示意

```
main.py
  │
  ├─ 🔴 USER_PROMPT_SUBMIT
  │
  └─ agent_loop(state, config)
       │
       ├─ run_one_turn()
       │    └─ execute_tool_use_blocks()
       │         ├─ 🟡 PRE_TOOL_USE  ×N
       │         ├─ handler() + L1
       │         └─ 🟢 POST_TOOL_USE ×N
       │
       └─ 🔵 STOP  ← 统一退出点

delegate_task (子 agent 派发)
  ├─ 🟣 SUBAGENT_START
  ├─ agent_loop(child_state, SUBAGENT_CONFIG)  ← 复用同一套 hook
  └─ 🟤 SUBAGENT_STOP
```

### 4.3 全局性说明

子 agent 通过 `agent_loop(child_state, config=SUBAGENT_CONFIG)` 派发，与父 agent 共享同一条 `agent_loop()` → `run_one_turn()` → `execute_tool_use_blocks()` 调用链。因此 **PRE_TOOL_USE、POST_TOOL_USE、STOP 天然对子 agent 也生效**。

hook 内通过 `ctx.config.role` 区分当前是 `"parent"` 还是 `"subagent"`：

```python
def my_hook(ctx):
    if ctx.config.role == "subagent":
        return  # 子 agent 中跳过
    # ... 父 agent 逻辑
```

USER_PROMPT_SUBMIT 不在 `agent_loop` 调用链上，因此**不对子 agent 触发**。

## 5. 核心 API 设计

### 5.1 HookContext

```python
@dataclass
class HookContext:
    stage: HookStage          # 当前阶段
    state: LoopState          # agent 状态（可读写）
    config: AgentConfig       # agent 配置（frozen，只读；通过 .role 区分 parent/subagent）
    data: dict                # 阶段特定数据（hook 可修改）
```

### 5.2 Hook 函数签名

```python
HookFunc = Callable[[HookContext], HookContext | None | bool]
```

返回值语义：

| 返回值 | 含义 | 适用阶段 |
|--------|------|----------|
| `None` | 正常继续，上下文不变 | 所有 |
| `HookContext` | 上下文已修改，使用修改后的版本 | 所有 |
| `False` | **拦截/中止** | 仅 PRE_TOOL_USE |

### 5.3 HookRegistry

```python
class HookRegistry:
    def register(self, stage: HookStage, func: HookFunc,
                 priority: int = 100, critical: bool = False) -> None: ...

    def fire(self, stage: HookStage, ctx: HookContext) -> HookContext | bool: ...

    def clear(self) -> None: ...  # 清除所有 hook（测试用）
```

- `priority`：越小越先执行，默认 100
- `critical`：True 时 hook 异常会向上抛出（中断流程），False 时静默吞掉并打印日志

### 5.4 全局单例

```python
# hook_system.py 模块底部
hook_registry = HookRegistry()
```

使用方式参考 `register_tools.py` 中的 `TOOL_HANDLERS`：

```python
from learn_agent.hook_system import HookStage, hook_registry

def my_logging_hook(ctx):
    print(f"[{ctx.stage.value}] tool={ctx.data.get('tool_name')}")
    return None

hook_registry.register(HookStage.POST_TOOL_USE, my_logging_hook, priority=50)
```

## 6. 各阶段详细设计

### 6.1 USER_PROMPT_SUBMIT

**注入点**：`main.py`，`agent_loop(state)` 调用前

```python
ctx = HookContext(
    stage=HookStage.USER_PROMPT_SUBMIT,
    state=state,
    config=PARENT_AGENT_CONFIG,
    data={"prompt": query},
)
result = hook_registry.fire(HookStage.USER_PROMPT_SUBMIT, ctx)
if isinstance(result, HookContext):
    # hook 可能修改了 prompt
    query = result.data.get("prompt", query)
```

**典型用途**：日志记录用户输入、prompt 预处理、敏感词过滤。

### 6.2 PRE_TOOL_USE

**注入点**：`agent_loop.py`，`execute_tool_use_blocks()`，handler 调用前

```python
ctx = HookContext(
    stage=HookStage.PRE_TOOL_USE,
    state=state,
    config=config,
    data={"tool_name": name, "tool_input": dict(tool_input)},
)
result = hook_registry.fire(HookStage.PRE_TOOL_USE, ctx)
if result is False:
    # 工具被拦截
    results.append({
        "type": "tool_result",
        "tool_use_id": tu["id"],
        "content": f"Error: tool '{name}' was blocked by PreToolUse hook. "
                   f"Reason: a hook returned False to prevent execution. "
                   f"Please try a different approach.",
    })
    continue

# Hook 可能修改了 tool_input
if isinstance(result, HookContext):
    tool_input = result.data.get("tool_input", tool_input)
```

**拦截时返回的信息**：告知模型工具被阻止，让模型知道发生了什么并尝试其他方法。

**典型用途**：危险命令拦截、工具调用频率限制、参数校验。

### 6.3 POST_TOOL_USE

**注入点**：`agent_loop.py`，`execute_tool_use_blocks()`，handler 返回 + L1 compaction 之后

```python
# ── L1 compaction ──（已有逻辑）
if config.has_compaction_layer("L1"):
    output, l1_applied = apply_l1_compaction(output, name, tool_input, state)

# ── PostToolUse hook ──
ctx = HookContext(
    stage=HookStage.POST_TOOL_USE,
    state=state,
    config=config,
    data={"tool_name": name, "tool_input": tool_input, "result": output},
)
result = hook_registry.fire(HookStage.POST_TOOL_USE, ctx)
if isinstance(result, HookContext):
    output = result.data.get("result", output)
```

**典型用途**：结果日志、统计分析、工具输出后处理、安全审计。未来 L1 compaction 可以迁移为 POST_TOOL_USE hook。

### 6.4 SUBAGENT_START

**注入点**：`tools/run_delegate_task.py`，`agent_loop(child_state, ...)` 调用前

```python
ctx = HookContext(
    stage=HookStage.SUBAGENT_START,
    state=state,           # 父 agent 的 state
    config=SUBAGENT_CONFIG,
    data={
        "task": task,
        "context": context,
        "relevant_paths": relevant_paths or [],
    },
)
result = hook_registry.fire(HookStage.SUBAGENT_START, ctx)
if isinstance(result, HookContext):
    task = result.data.get("task", task)
    relevant_paths = result.data.get("relevant_paths", relevant_paths)
```

**典型用途**：子 agent 派发日志、任务内容监控、子 agent 数量限制。

### 6.5 SUBAGENT_STOP

**注入点**：`tools/run_delegate_task.py`，`agent_loop()` 返回后

```python
# agent_loop 已返回
child_result = _extract_final_text(child_state)

ctx = HookContext(
    stage=HookStage.SUBAGENT_STOP,
    state=state,           # 父 agent 的 state
    config=SUBAGENT_CONFIG,
    data={
        "result": child_result,
        "child_state": child_state,
    },
)
result = hook_registry.fire(HookStage.SUBAGENT_STOP, ctx)
if isinstance(result, HookContext):
    child_result = result.data.get("result", child_result)
```

**典型用途**：子 agent 结果审核、子 agent 执行统计、调试日志。

### 6.6 STOP

**注入点**：`agent_loop.py`，所有退出路径统一到此

```python
def agent_loop(state, config):
    while state.turn_count <= config.max_turns:
        should_continue = run_one_turn(state, config)
        if not should_continue:
            break  # 改为 break，不再直接 return

    # 处理 max_turns_exceeded
    if state.turn_count > config.max_turns and not state.stopped_reason:
        state.stopped_reason = "max_turns_exceeded"
        append_safety_stop_message(state, "max_turns_exceeded")

    # ── STOP hook（统一退出点）──
    reason = state.stopped_reason or "normal"
    ctx = HookContext(
        stage=HookStage.STOP,
        state=state,
        config=config,
        data={"reason": reason},
    )
    hook_registry.fire(HookStage.STOP, ctx)
```

`reason` 取值：

| 值 | 含义 |
|----|------|
| `"normal"` | 模型返回纯文本，自然结束 |
| `"max_turns"` | 达到最大轮数上限 |
| `"max_failures"` | 达到最大失败次数 |

**典型用途**：执行统计摘要、资源清理、transcript 收尾。

## 7. 实现说明

### 7.1 新增文件

```
src/learn_agent/hook_system.py    # HookStage, HookContext, HookRegistry, hook_registry
src/learn_agent/hooks/__init__.py # 内置 hook（暂为空，预留）
```

### 7.2 修改文件

| 文件 | 修改内容 |
|------|----------|
| `agent_loop.py` | `agent_loop()` 改为 break + 统一退出点（STOP hook）；`execute_tool_use_blocks()` 注入 PRE_TOOL_USE + POST_TOOL_USE |
| `main.py` | 注入 USER_PROMPT_SUBMIT |
| `tools/run_delegate_task.py` | 注入 SUBAGENT_START + SUBAGENT_STOP |

### 7.3 执行顺序

在 `execute_tool_use_blocks()` 中，每个工具的执行顺序为：

```
1. PRE_TOOL_USE hook（可拦截）
2. tool handler 执行
3. L1 compaction（现有逻辑）
4. POST_TOOL_USE hook（可修改 result）
```

### 7.4 错误处理

```python
def fire(self, stage, ctx):
    for priority, hook, critical in self._hooks[stage]:
        try:
            result = hook(ctx)
            if result is False:
                return False
            elif isinstance(result, HookContext):
                ctx = result
        except Exception as e:
            print(f"[Hook] {stage.value} error in hook (priority={priority}): {e}")
            if critical:
                raise
    return ctx
```

- 非 critical hook 异常 → 打印日志，继续
- critical hook 异常 → 向上抛出，中断流程

## 8. 配置项

Hook 系统本身不需要配置项。单个 hook 的配置由 hook 函数自行管理（从 settings 读取或硬编码）。

## 9. 使用示例

```python
"""示例：工具调用日志 hook"""
from learn_agent.hook_system import HookStage, HookContext, hook_registry

def log_tool_calls(ctx: HookContext) -> HookContext | None:
    tool_name = ctx.data.get("tool_name", "?")
    if ctx.stage == HookStage.PRE_TOOL_USE:
        print(f"[LOG] → {tool_name}({ctx.data.get('tool_input', {})})")
    elif ctx.stage == HookStage.POST_TOOL_USE:
        result_len = len(ctx.data.get("result", ""))
        print(f"[LOG] ← {tool_name} returned {result_len} chars")
    return None

# 注册
hook_registry.register(HookStage.PRE_TOOL_USE, log_tool_calls, priority=10)
hook_registry.register(HookStage.POST_TOOL_USE, log_tool_calls, priority=10)
```

```python
"""示例：阻止执行 rm -rf"""
def block_dangerous_commands(ctx: HookContext) -> HookContext | bool:
    if ctx.data.get("tool_name") != "bash":
        return None
    cmd = ctx.data.get("tool_input", {}).get("command", "")
    dangerous = ["rm -rf /", "sudo rm", "shutdown", "reboot"]
    if any(d in cmd for d in dangerous):
        return False  # 拦截
    return None

hook_registry.register(HookStage.PRE_TOOL_USE, block_dangerous_commands, priority=5)
```

```python
"""示例：父 agent 退出时打印摘要（子 agent 跳过）"""
def exit_summary(ctx: HookContext) -> HookContext | None:
    if ctx.config.role != "parent":
        return None
    reason = ctx.data.get("reason", "?")
    turns = ctx.state.turn_count
    failures = ctx.state.failure_count
    print(f"\n[SUMMARY] Agent exited ({reason}): {turns} turns, {failures} failures")
    return None

hook_registry.register(HookStage.STOP, exit_summary, priority=100)
```

## 10. 测试要点

1. Hook 注册和触发顺序（按 priority）
2. Hook 返回 `None` 时流程正常继续
3. Hook 返回 `HookContext` 时数据被修改且后续 hook 看到修改后的值
4. PRE_TOOL_USE 返回 `False` 时工具被拦截，模型收到 error 信息
5. Hook 异常不中断流程（非 critical）
6. critical hook 异常正确抛出
7. 子 agent 派发时 SUBAGENT_START / SUBAGENT_STOP 正确触发
8. 子 agent 内部 PRE_TOOL_USE / POST_TOOL_USE / STOP 也正确触发
9. `hook_registry.clear()` 清除所有 hook
