# 技能系统设计方案

日期：2026-06-15  
项目：`learn-agent`  
文档状态：设计确认稿，等待 review

## 1. 背景

当前项目已具备完整的 Agent Loop、工具系统、计划系统和子代理机制。但模型面对不同类型任务时，缺少可复用的"领域知识包"——例如调试时应该遵循什么流程、写测试时应该采用什么策略。

技能系统（Skills）解决的就是这个问题：将特定领域的操作指南打包为可发现、可加载的文档，模型按需获取。

## 2. 目标

1. 项目启动时自动发现 `.agents/skills/` 下所有可用技能。
2. 将技能摘要（名称 + 描述）注入 system prompt，让模型知道有哪些技能可用。
3. 提供 `load_skill` 工具，模型决定需要某个技能时主动加载其正文。
4. 技能正文作为 `tool_result` 返回，不占据 system prompt 空间。
5. 仅父 agent 可使用 `load_skill`，子 agent 无权限。
6. 文件大小保护：明确约定而非代码截断。

## 3. 非目标

1. 不自动加载技能（模型按需决定）。
2. 不把技能正文放入 system prompt。
3. 不支持技能的动态热加载（需重启程序）。
4. 不做技能间的依赖关系。
5. 不引入第三方 YAML/Markdown 解析库。
6. 不对 SKILL.md 做内容截断。

## 4. 技能目录结构

```
.agents/skills/                      ← 可配置的根目录
├── test-driven-development/
│   ├── SKILL.md                     ← 必需：技能定义文件
│   └── testing-anti-patterns.md     ← 可选：附属参考文件
├── systematic-debugging/
│   ├── SKILL.md
│   ├── root-cause-tracing.md
│   ├── condition-based-waiting.md
│   └── find-polluter.sh
├── using-git-worktrees/
│   └── SKILL.md
└── ...
```

**规则：**

- 每个技能一个子目录，目录名即技能标识。
- 目录下必须包含 `SKILL.md`。
- 目录下可包含任意附属文件（Markdown、脚本、模板等），由模型的 `read_file` / `glob` 工具自然发现和加载。

## 5. SKILL.md 格式

```markdown
---
name: test-driven-development
description: Use when implementing any feature or bugfix, before writing implementation code
---

# Test-Driven Development (TDD)

## Overview

Write the test first. Watch it fail. Write minimal code to pass.

## When to Use

- New features
- Bug fixes
- Refactoring

## Process

1. Write a failing test...
2. Run to confirm failure...
3. Implement minimal code...
```

### 5.1 Frontmatter

- 以 `---` 开始和结束的 YAML 块。
- 仅解析 `name` 和 `description` 两个字段。
- 解析失败（缺失或格式错误）→ 该技能被跳过，打印警告。

### 5.2 Body

- Frontmatter 之后的所有内容为正文。
- 正文是标准的 Markdown，可包含标题、列表、代码块等。
- 正文由 `load_skill` 工具完整返回给模型。
- **不做截断**。遵循最佳实践：SKILL.md 保持精简（< 500 行），充当导航文档。模型读完 SKILL.md 后如需深入了解，使用 `read_file` 自行读取附属文件。

## 6. 数据结构

### 6.1 `SkillInfo`

```python
@dataclass
class SkillInfo:
    name: str           # 技能名称
    description: str    # 一句话描述（出现在 system prompt 中）
    body: str           # SKILL.md 正文全文
    dir_path: str       # 技能目录的绝对路径（方便 read_file 定位附属文件）
```

### 6.2 `SkillRegistry`

```python
class SkillRegistry:
    def __init__(self, skills_root: str = ".agents/skills") -> None: ...
    def discover(self) -> int: ...
    def build_skills_prompt(self) -> str: ...
    def load_skill(self, name: str) -> str | None: ...
```

**职责：**

| 方法 | 何时调用 | 行为 |
|------|---------|------|
| `__init__` | 程序启动 | 记录根目录路径 |
| `discover()` | 程序启动后立即 | 扫描 `*/SKILL.md`，解析 frontmatter，构建内存索引 |
| `build_skills_prompt()` | 每次 `build_system` | 返回技能摘要列表字符串，注入 system prompt |
| `load_skill(name)` | 模型调用 tool 时 | 返回技能正文全文，未找到返回 `None` |

## 7. 发现流程（`discover`）

```
discover()
  1. glob(".agents/skills/*/SKILL.md")
  2. 对每个匹配的 SKILL.md：
     a. 读取文件
     b. 解析 frontmatter（手动逐行解析）
        - 第一行必须是 "---"
        - 持续读取直到下一个 "---"
        - 按 "key: value" 拆分行
        - 提取 name 和 description
     c. 解析失败 → 打印 ⚠ 警告，跳过
     d. 解析成功 → 提取 body（第二个 --- 之后的内容）
     e. 存入 SkillInfo 列表
  3. 返回发现的有效技能数量
```

**解析规则：**

```python
def _parse_frontmatter(text: str) -> tuple[dict, str]:
    """Returns ({name, description}, body_after_frontmatter) or raises ValueError."""
    lines = text.split("\n")
    if not lines or lines[0].strip() != "---":
        raise ValueError("Missing opening ---")
    
    end_idx = None
    for i in range(1, len(lines)):
        if lines[i].strip() == "---":
            end_idx = i
            break
    
    if end_idx is None:
        raise ValueError("Missing closing ---")
    
    frontmatter = {}
    for line in lines[1:end_idx]:
        if ":" in line:
            key, _, value = line.partition(":")
            frontmatter[key.strip()] = value.strip()
    
    if "name" not in frontmatter or "description" not in frontmatter:
        raise ValueError("Missing name or description")
    
    body = "\n".join(lines[end_idx + 1:])
    return frontmatter, body
```

**仅父 agent 注入**。子 agent 不显示技能列表（也不在子 agent 的 allowed tools 中）。

## 9. `load_skill` 工具设计

### 9.1 工具定义

```json
{
  "name": "load_skill",
  "description": "Load a skill's full documentation. Skills provide domain-specific guidance for tasks like debugging, testing, etc.",
  "input_schema": {
    "type": "object",
    "properties": {
      "name": {
        "type": "string",
        "description": "Name of the skill to load"
      }
    },
    "required": ["name"]
  }
}
```

### 9.2 处理器

```python
# run_load_skill.py

def run_load_skill(state: LoopState, name: str) -> str:
    body = registry.load_skill(name)
    if body is None:
        available = ", ".join(s.name for s in registry.skills)
        return f"Error: skill '{name}' not found. Available: {available}"
    return body
```

### 9.3 权限

- `load_skill` 仅出现在父 agent 的 `allowed_tool_names` 中。
- 子 agent 不可调用（权限校验在 `execute_tool_use_blocks` 中已有）。

## 10. 文件大小策略

**结论：不做代码级截断。**

理由：

| 因素 | 说明 |
|------|------|
| 业界约定 | Anthropic 官方建议 SKILL.md < 500 行，写成长导航文档而非知识 dump |
| 现有技能 | 项目中 9 个 SKILL.md 均在合理范围内（最大约 200 行） |
| 模型能力 | 读完导航文档后，模型可用 `read_file` 按需加载附属文件 |
| 维护成本 | 开发者有责任保持 SKILL.md 精简，代码层面不做额外保护 |
| 未来扩展 | 如果真的需要保护，可在 `discover()` 阶段统计行数并打印警告 |

不做截断不是偷懒，而是认为 SKILL.md 的大小问题应该在编写阶段解决，而不是运行时硬砍。

## 12. 测试计划

### 12.1 发现

- 扫描真实 `.agents/skills/` 目录，验证至少发现 N 个技能。
- 模拟临时目录包含合法/不合法的 SKILL.md，验证解析正确性。
- Frontmatter 缺失 name → 跳过并警告。
- Frontmatter 缺失 closing `---` → 跳过并警告。
- 空目录无 SKILL.md → 返回 0。

### 12.2 build_skills_prompt

- 有技能时返回摘要列表。
- 无技能时返回空字符串（不影响 system prompt）。

### 12.3 load_skill

- 已知技能名 → 返回正文。
- 未知技能名 → 返回 "not found" 错误，列出可用技能名。

### 12.4 权限

- 子 agent 调用 `load_skill` → 权限错误。
- 父 agent 调用 `load_skill` → 正常返回。

### 12.5 集成

- System prompt 在启动时包含技能列表。
- 模型调用 `load_skill` 后，正文出现在 messages 中。
- 回归：现有 142 个测试全过。
